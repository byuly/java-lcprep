Trie (prefix tree)
O(m) per op▼
Build from scratch in interviews. Used for: autocomplete, word search II, prefix matching, IP routing.
class Trie {
    Trie[] ch = new Trie[26];
    boolean end;

    public void insert(String word) {
        Trie n = this;
        for(char c : word.toCharArray()) {
            int i = c - 'a';
            if(n.ch[i] == null)
                n.ch[i] = new Trie();
            n = n.ch[i];
        }
        n.end = true;
    }

    public boolean search(String word) {
        Trie n = get(word);
        return n != null && n.end;
    }

    public boolean startsWith(String p) {
        return get(p) != null;
    }

    private Trie get(String s) {
        Trie n = this;
        for(char c : s.toCharArray()) {
            if((n = n.ch[c-'a']) == null)
                return null;
        }
        return n;
    }
}
Binary search
O(log n)▼
Safe mid: always l + (r-l)/2 to prevent overflow. Know all three variants.
Exact match
int l=0, r=nums.length-1;
while(l<=r) {
    int mid=l+(r-l)/2;
    if(nums[mid]==target) return mid;
    if(nums[mid]<target) l=mid+1;
    else r=mid-1;
}
return -1;
Left boundary (first ≥ target)
int l=0, r=nums.length;
while(l<r) {
    int mid=l+(r-l)/2;
    if(nums[mid]<target) l=mid+1;
    else r=mid;
}
return l;
Binary search on answer (most powerful template)
// "Find min X such that condition(X) is true"
// Used in: Koko bananas, split array, ship packages
int l=minVal, r=maxVal;
while(l<r) {
    int mid=l+(r-l)/2;
    if(condition(mid)) r=mid;
    else l=mid+1;
}
return l;
Tree traversals
O(n)▼
All 3 DFS recursive
// Inorder: L→Root→R → sorted BST output
void inorder(TreeNode n) {
    if(n==null) return;
    inorder(n.left);
    process(n);
    inorder(n.right);
}
// Preorder: Root→L→R → serialize / copy
// Postorder: L→R→Root → delete / heights
Iterative inorder
Deque<TreeNode> st=new ArrayDeque<>();
TreeNode cur=root;
while(cur!=null||!st.isEmpty()) {
    while(cur!=null) {
        st.push(cur); cur=cur.left;
    }
    cur=st.pop();
    process(cur);
    cur=cur.right;
}
BFS — level order (key: snapshot level size)
Queue<TreeNode> q=new LinkedList<>();
q.offer(root);
while(!q.isEmpty()) {
    int sz=q.size(); // snapshot = exactly one level
    for(int i=0;i<sz;i++) {
        TreeNode n=q.poll();
        if(n.left!=null) q.offer(n.left);
        if(n.right!=null) q.offer(n.right);
    }
}

DFS grid (4 directions)
int[][] dirs={{0,1},{0,-1},{1,0},{-1,0}};

void dfs(int[][] g, int r, int c) {
    g[r][c]=0; // mark visited in-place
    for(int[] d:dirs) {
        int nr=r+d[0], nc=c+d[1];
        if(nr>=0&&nr<g.length&&
           nc>=0&&nc<g[0].length&&
           g[nr][nc]==1)
            dfs(g,nr,nc);
    }
}

Union Find
O(α(n)) ≈ O(1)▼
Always use path compression + union by rank. Used for: connected components, cycle detection, redundant connections, number of provinces.
int[] parent, rank;
void init(int n){
    parent=new int[n]; rank=new int[n];
    for(int i=0;i<n;i++) parent[i]=i;
}
int find(int x){          // path compression
    if(parent[x]!=x) parent[x]=find(parent[x]);
    return parent[x];
}
boolean union(int a, int b){
    int pa=find(a), pb=find(b);
    if(pa==pb) return false; // already connected
    if(rank[pa]<rank[pb]) parent[pa]=pb;
    else if(rank[pa]>rank[pb]) parent[pb]=pa;
    else{ parent[pb]=pa; rank[pa]++; }
    return true;
}
Sliding window & two pointers
O(n)▼
Variable window
int left=0, res=0;
for(int right=0; right<n; right++) {
    // expand: include nums[right]
    while(invalid condition) {
        // shrink: remove nums[left]
        left++;
    }
    res=Math.max(res, right-left+1);
}
Fixed window
// Max sum subarray of size k
int sum=0, res=0;
for(int i=0; i<nums.length; i++) {
    sum+=nums[i];
    if(i>=k) sum-=nums[i-k];
    if(i>=k-1) res=Math.max(res,sum);
}
Two pointers (sorted array)
int l=0, r=nums.length-1;
while(l<r) {
    int s=nums[l]+nums[r];
    if(s==target){ l++; r--; }
    else if(s<target) l++;
    else r--;
}
Backtracking
O(n!) worst▼
Pattern: choose → explore → unchoose. Always copy the list: new ArrayList<>(cur).
List<List<Integer>> res=new ArrayList<>();

void bt(int start, List<Integer> cur) {
    if(baseCase) {
        res.add(new ArrayList<>(cur)); // ALWAYS copy!
        return;
    }
    for(int i=start; i<nums.length; i++) {
        if(skipCondition) continue; // pruning
        cur.add(nums[i]);         // choose
        bt(i+1, cur);             // explore
        cur.remove(cur.size()-1); // unchoose
    }
}
// Permutations: use boolean[] used, pass i=0
// Combinations: pass start index (i+1)
// Subsets: add cur at EVERY level, no base case
// Dedupe: sort first, skip i>start && nums[i]==nums[i-1]
Monotonic stack
O(n)▼
Used for: next greater/smaller element, daily temperatures, largest rectangle in histogram, trapping rain water. Each element pushed/popped at most once → O(n).
Next greater (decreasing stack)
int[] res=new int[n];
Arrays.fill(res,-1);
Deque<Integer> st=new ArrayDeque<>();
for(int i=0;i<n;i++){
    while(!st.isEmpty()&&
          nums[st.peek()]<nums[i])
        res[st.pop()]=nums[i];
    st.push(i);
}
Next smaller (increasing stack)
int[] res=new int[n];
Arrays.fill(res,-1);
Deque<Integer> st=new ArrayDeque<>();
for(int i=0;i<n;i++){
    while(!st.isEmpty()&&
          nums[st.peek()]>nums[i])
        res[st.pop()]=nums[i];
    st.push(i);
}

Floyd's cycle detection (fast/slow)
O(n) · O(1) space▼
// Phase 1: detect cycle
ListNode slow=head, fast=head;
while(fast!=null&&fast.next!=null){
    slow=slow.next; fast=fast.next.next;
    if(slow==fast) break;
}
if(fast==null||fast.next==null) return null; // no cycle

// Phase 2: find cycle entry node
slow=head;
while(slow!=fast){
    slow=slow.next; fast=fast.next;
}
return slow; // cycle entry

Common pitfalls & interview mistakes
watch out▼
⚠Never compare Strings with ==. Use .equals(). Integer cache only covers -128 to 127, so == fails unpredictably.
⚠Comparator (a,b) -> a-b overflows with negatives. Use Integer.compare(a,b) or a < b ? -1 : a > b ? 1 : 0.
⚠Backtracking: res.add(cur) adds a reference that gets cleared. Always write res.add(new ArrayList<>(cur)).
⚠ConcurrentModificationException: don't remove from a list while iterating. Collect indices first, or use Iterator.remove().
⚠Integer overflow in multiplication: cast first → (long)a * b. Without cast, overflow happens before the assignment.
⚠Binary search: mixing l <= r and l < r causes infinite loops or missed values. Pick one invariant and stick to it.
⚠map.get(key) returns null when key is absent. Auto-unboxing to int throws NPE. Always use getOrDefault(k, 0).
⚠Always check node != null before .val, .next, .left/.right. Check fast.next != null too in Floyd's.
⚠Queue/Deque: poll() and peek() return null on empty. pop() and remove() throw exceptions. Prefer poll()/peek().
